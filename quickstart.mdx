title: "Quickstart"
description: "Install jmp, declare your first static branch, and run a working example."
---

<Steps>
  <Step title="Install the header">
    `jmp` is a single header file with no dependencies. Download it directly into your project:

    <CodeGroup>

    ```bash wget
    wget https://raw.githubusercontent.com/qlibs/jmp/main/jmp
    ```

    ```bash curl
    curl -O https://raw.githubusercontent.com/qlibs/jmp/main/jmp
    ```

    </CodeGroup>

    Alternatively, if you are using C++20 modules, download the module interface unit:

    <CodeGroup>

    ```bash wget
    wget https://raw.githubusercontent.com/qlibs/jmp/main/jmp.cppm
    ```

    ```bash curl
    curl -O https://raw.githubusercontent.com/qlibs/jmp/main/jmp.cppm
    ```

    </CodeGroup>
  </Step>

  <Step title="Include the header">
    Add the header to your source file:

    ```cpp
    #include "jmp"
    ```

    Or, for module-based builds:

    ```cpp
    import jmp;
    ```

    <Note>
      By default, `jmp` runs compile-time self-tests (`static_assert`) on every include to verify the library works correctly with your compiler and environment. To disable these tests, compile with `-DNTEST`.
    </Note>
  </Step>

  <Step title="Declare a static branch">
    Declare a `static_branch` as a `constexpr` global or static variable. The initial value must match the branch's default state:

    ```cpp
    // Boolean branch — must be initialized to false
    constexpr jmp::static_branch<bool> static_bool = false;

    // Integer range branch — must be initialized to Min (0 here)
    // Range constraint: Max - Min must be >= 2 and <= 7
    constexpr jmp::static_branch<int, 0, 2> static_int = 0;
    ```

    <Warning>
      `static_branch<bool>` must be initialized to `false`. `static_branch<T, Min, Max>` must be initialized to `Min`. Initializing to any other value is a compile-time error.
    </Warning>

    Copy and move constructors and assignment operators are deleted — static branches are unique objects. You cannot copy or move them.
  </Step>

  <Step title="Initialize at program start">
    Before changing any branch value, call `jmp::init()` once at program startup. This uses `mprotect` to make the code pages writable so that subsequent assignments can patch the instructions in memory.

    ```cpp
    int main() {
      if (not jmp::init()) {  // makes code pages writable for patching
        return errno;
      }

      // safe to assign branches from here on
    }
    ```

    `jmp::init()` returns `true` on success. On failure it returns `false` and sets `errno`.
  </Step>

  <Step title="Use and update the branch">
    Use a `static_branch<bool>` in an `if` statement just like a regular bool. Assign to it like any variable to switch the branch at runtime:

    ```cpp
    #include "jmp"
    #include <cstdio>
    #include <cerrno>

    constexpr jmp::static_branch<bool> static_bool = false;

    // Note: fn can be inline, noinline, constexpr, etc.
    void fn() {
      if (static_bool) {        // [[likely]] / [[unlikely]] have no impact
        std::puts("taken");
      } else {
        std::puts("not taken");
      }
    }

    int main() {
      if (not jmp::init()) {
        return errno;
      }

      fn();               // prints: not taken

      static_bool = true; // patches nop -> jmp in memory
      fn();               // prints: taken
    }
    ```

    The compiler generates the following assembly (with `-O3`):

    ```asm
    main:
    # init
      ...

    # fn() — not taken
      nop               ; static_bool is false: single nop, falls through
    .Ltmp0:
      mov edi, OFFSET FLAT:.LC0
      jmp puts
      ret
    .Ltmp1:
      mov edi, OFFSET FLAT:.LC1
      jmp puts
      ret

    # static_bool = true
      call static_bool.operator=(true)

    # fn() — taken
      jmp .Ltmp1        ; nop has been patched to jmp .Ltmp1 in memory
    .Ltmp0:
      mov edi, OFFSET FLAT:.LC0
      jmp puts
      ret
    .Ltmp1:
      mov edi, OFFSET FLAT:.LC1
      jmp puts
      ret

    .LC0: .asciz "not taken"
    .LC1: .asciz "taken"
    ```

    <Tip>
      If you mark `fn` with `[[gnu::noinline]]`, the patching cost when you assign a new value is a single `memcpy` of 5 bytes (for `static_branch<bool>`) or 4 bytes (for `static_branch<T, Min, Max>`). If `fn` is inlined at multiple call sites, the cost scales linearly: one `memcpy` per inlined instance. Use `[[gnu::noinline]]` on hot-path functions to keep switching cost constant and predictable.
    </Tip>
  </Step>
</Steps>

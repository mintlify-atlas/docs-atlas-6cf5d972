`jmp` is a C++20 library that implements **static branches** — branches whose condition is stable at runtime and can be patched directly in machine code. Instead of a CPU branch or memory read on every call, `jmp` emits a single `nop` or `jmp` instruction that is rewritten in-place when the condition changes.

This technique is used in the Linux kernel under the name [static keys](https://docs.kernel.org/staging/static-keys.html) and is ideal for high-performance paths where branch misprediction or cache pressure is a bottleneck.

## Why `jmp`?

Standard C++ branches — `if`, `switch`, ternary — require the CPU to evaluate a condition on every execution. When that condition is:

- **Stable** — rarely changes through a program's lifetime
- **Expensive** — requires a memory load or cache miss to evaluate
- **Hard to predict** — confuses the branch predictor

...the conventional approach wastes cycles. `jmp` solves this by replacing the branch with a single `nop` (no-operation) instruction when the condition is false, and patching it to a `jmp` when it becomes true. The CPU never needs to evaluate the condition again until it changes.

<Steps>
  <Step title="Include the header">
    Copy `jmp` (single header) or use the C++20 module `jmp.cppm` into your project. No build system changes required.
  </Step>
  <Step title="Declare a static branch">
    ```cpp
    constexpr jmp::static_branch<bool> logging_enabled = false;
    ```
  </Step>
  <Step title="Enable code patching at startup">
    ```cpp
    if (not jmp::init()) { return errno; }
    ```
  </Step>
  <Step title="Use and update the branch at runtime">
    ```cpp
    if (logging_enabled) { log("..."); }

    // Later, switch it on — zero-cost from this point forward:
    logging_enabled = true;
    ```
  </Step>
</Steps>

<Note>
`jmp` currently supports **x86-64 Linux** with clang++ 10+ or g++ 10+. The technique is architecture-portable but only x86-64 is implemented.
</Note>
